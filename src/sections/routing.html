Introduction to React Router

What is routing? Imagine a website as a book. In regular websites, when you click a link, it's like replacing the entire book just to read a different page. But with React Router, it's like flipping to a new page while keeping the same book in your hands - much faster and smoother! ✨

Simple Definition

React Router is a tool that helps your React app show different content when the user clicks on different links, without reloading the entire page.

As a beginner, here's what React Router does for your app:

  • Shows different screens when the URL changes (like showing a profile page when you go to /profile)
  • Changes the URL in your browser without refreshing the page (making navigation feel instant)
  • Keeps everything in sync so the URL always matches what's on screen

The big benefit? Your app feels like a smooth, native app instead of a clunky website. Users can navigate around without waiting for new pages to load!

Real-world example: Think of how apps like Instagram or Twitter let you click on profiles and posts without the whole page refreshing. That smooth navigation is what React Router brings to your web apps!

Core Router Components Explained

React Router has three main components that work together to make routing work: BrowserRouter, Routes, and Route. Let's understand what each one does:

BrowserRouter

Think of it as: The foundation of your routing system

BrowserRouter is the main container for all your routing. It uses the HTML5 history API to keep your UI in sync with the URL in the browser address bar.

What it does:

  • Listens for URL changes
  • Manages browser history
  • Enables back/forward button functionality
  • Passes routing information to child components

Routes

Think of it as: A switchboard that directs traffic

The Routes component looks at all of its child Route components and renders the one whose path best matches the current URL. It ensures only one route is shown at a time.

What it does:

  • Groups all your Route components
  • Finds the best matching Route for the current URL
  • Prevents multiple routes from rendering at once
  • Replaced the old "Switch" component from React Router v5

Route

Think of it as: A single road to a destination

Each Route component maps a URL path to a specific React component. When the current URL matches its path, it renders the specified component.

What it does:

  • Connects a URL path to a React component
  • Takes a "path" prop that defines the URL pattern
  • Takes an "element" prop with the component to render
  • Can handle dynamic parameters with path patterns like "/users/:userId"

How They Work Together

These three components work together like a well-organized transportation system:

  1. BrowserRouter creates the environment for routing to work
  2. Routes acts like a traffic controller, looking at all available routes
  3. Route components define the individual paths and what should appear when those paths are visited

Together, they create a seamless navigation experience where the UI updates to match the URL without reloading the page.

Important: Always make sure your Route components are wrapped inside a Routes component, and that everything is inside a BrowserRouter. This structure is essential for React Router to work properly.

Basic Route Configuration

Setting up routes is like drawing a map for your app that says "when the user goes to this address, show this screen." Each route connects a URL path (like /about) to a specific component (like AboutPage).

Step 1

Import the necessary components from React Router

Step 2

Wrap your app in a BrowserRouter component

Step 3

Define your routes using the Routes and Route components

Here's a simple example showing how to create routes in your app:

App.jsx 
import { BrowserRouter, Routes, Route } from 'react-router-dom';

// Your page components
import Home from './pages/Home';
import About from './pages/About';
import Contact from './pages/Contact';

function App() {
    return (
        <BrowserRouter>            {/* Step 1: The router container */}
            <Routes>                {/* Step 2: Group all your routes */}
                {/* Step 3: Define each route with path and component */}
                <Route path="/" element={<Home />} />             
                {/* When URL is yoursite.com/ show the Home component */}
                
                <Route path="/about" element={<About />} />       
                {/* When URL is yoursite.com/about show the About component */}
                
                <Route path="/contact" element={<Contact />} />   
                {/* When URL is yoursite.com/contact show the Contact component */}
            </Routes>
        </BrowserRouter>
    );
}

Note: If you see code that looks different, it might be using an older version of React Router. Here's how it looked in v5: 


import { BrowserRouter as Router, Route, Switch } from 'react-router-dom';

function App() {
    return (
        <Router>
            <Switch>                {/* This is now called 'Routes' in v6 */}
                <Route path="/" exact component={Home} />      {/* In v6: element={<Home />} */}
                <Route path="/about" component={About} />
                <Route path="/contact" component={Contact} />
            </Switch>
        </Router>
    );
}

Navigation Components

Once you have routes set up, you need ways for users to navigate between them. React Router gives you special components that work like links but don't cause page refreshes.

Link Component

What's a Link?

In regular HTML, we use <a> tags for links, but these reload the entire page when clicked. The Link component from React Router looks like a regular link but performs navigation without page reloads!

Regular Link (Don't Use)

<a href="/about">About</a>

Causes full page reload!

React Router Link (Use This)

<Link to="/about">About</Link>

Smooth navigation without reload!

Here's how to create a navigation menu with Link components:

Navigation.jsx 
import { Link } from 'react-router-dom';  // Step 1: Import the Link component

function Navigation() {
    return (
        <nav className="bg-gray-100 p-4 rounded">
            <ul className="flex space-x-4">
                {/* Step 2: Replace all <a> tags with <Link> components */}
                <li><Link to="/" className="text-blue-500 hover:underline">Home</Link></li>
                <li><Link to="/about" className="text-blue-500 hover:underline">About</Link></li>
                <li><Link to="/contact" className="text-blue-500 hover:underline">Contact</Link></li>
                {/* Notice we use "to" instead of "href" */}
            </ul>
        </nav>
    );
}

Programmatic Navigation

Sometimes you need to navigate using code instead of links. For example, after a user submits a form or when a timer completes. That's when programmatic navigation comes in: 


import { useNavigate } from 'react-router-dom';

function LoginForm() {
    const navigate = useNavigate();
    
    const handleSubmit = (event) => {
        event.preventDefault();
        // Process login...
        
        // Redirect after successful login
        navigate('/dashboard');
        
        // You can also pass state to the new location
        // navigate('/dashboard', { state: { from: 'login' } });
    };
    
    return (
        <form onSubmit={handleSubmit}>
            {/* Form fields */}
            <button type="submit">Login</button>
        </form>
    );
}

Nested Routes

Nested routes let you create layouts with sub-pages. Think of them like folders inside folders: 


import { BrowserRouter, Routes, Route, Outlet } from 'react-router-dom';

function App() {

function Layout() {
    return (
        <div>
            <header>Header content</header>
            <main>
                {/* Nested routes are rendered here */}
                <Outlet />
            </main>
            <footer>Footer content</footer>
        </div>
    );
}

Dynamic Routes with Parameters

Dynamic Routes: One Route, Many Pages

Instead of creating separate routes for each item (like /products/shirt, /products/pants), dynamic routes let you create one flexible path with variable parts that change based on what the user wants to see.

How It Works

In the route /products/:id:

  • • :id is a variable that captures any value
  • • /products/123 → id = "123"
  • • /products/t-shirt → id = "t-shirt"
Perfect For
  • • Product pages
  • • User profiles
  • • Blog posts
  • • Any content with similar layouts but different data
3 Simple Steps: Dynamic Routes in Action 
// Step 1: Define a dynamic route with parameter
<Route path="/products/:id" element={<ProductDetail />} />

// Step 2: Create a component that uses the parameter
import { useParams } from 'react-router-dom';

function ProductDetail() {
    // Step 3: Extract the parameter value with useParams hook
    const { id } = useParams();
    
    return (
        <div>
            <h1>Product: {id}</h1>
            {/* Now you can use the id to fetch product data */}
        </div>
    );
}

// BONUS: Working with query parameters (e.g., ?color=red&size=large)
import { useSearchParams } from 'react-router-dom';

function ProductFilters() {
    const [searchParams, setSearchParams] = useSearchParams();
    const color = searchParams.get('color'); // Gets "red" from ?color=red
    
    return (
        <div>
            <p>Selected color: {color || "none"}</p>
            {/* Update filters with setSearchParams({...}) */}
        </div>
    );
}

Protected Routes

What are Protected Routes?

Protected routes are pages in your app that only certain users should be able to see. Think of them like VIP rooms that require a special pass to enter. If someone tries to access these routes without permission, they get redirected elsewhere (usually to a login page).

Common examples of protected routes:

  • User dashboard
  • Profile settings
  • Account management pages
  • Admin panels

Creating protected routes is easy! Let's break it down into simple steps:

ProtectedRoutes.jsx 
import { Navigate, Outlet } from 'react-router-dom';

// Step 1: Create a component that will check if the user is logged in
function ProtectedRoute({ isAuthenticated }) {
    // Step 2: Check if user is authenticated
    if (!isAuthenticated) {
        // Step 3: If not authenticated, redirect to login
        return <Navigate to="/login" replace />;
        // The 'replace' prop replaces the current URL in history
        // so the user can't press the back button to access the protected page
    }
    
    // Step 4: If authenticated, render whatever is inside this route
    return <Outlet />;  // Outlet renders child routes
}

// Step 5: Use the ProtectedRoute in your route configuration
function AppRoutes() {
    // This would typically come from your auth system
    const isUserLoggedIn = checkIfUserIsLoggedIn();  
    
    return (
        <Routes>
            {/* Public routes anyone can access */}
            <Route path="/" element={<Home />} />
            <Route path="/login" element={<Login />} />
            
            {/* Protected routes - wrapped in our ProtectedRoute component */}
            <Route element={<ProtectedRoute isAuthenticated={isUserLoggedIn} />}>
                <Route path="/dashboard" element={<Dashboard />} />
                <Route path="/profile" element={<Profile />} />
                <Route path="/settings" element={<Settings />} />
            </Route>
        </Routes>
    );
}

Understanding the Outlet Component

You may have noticed the <Outlet /> component in the code above. This is a key part of how protected routes work:

What is Outlet?

Outlet is like a window or placeholder that shows the child routes within a parent route. Think of it as saying "if the user passes the authentication check, display whatever child route they're trying to access right here."

Why use Outlet?
  • • Enables nested routing in React Router
  • • Allows multiple routes to share the same protection logic
  • • Creates cleaner, more organized routing code
  • • Reduces code duplication for access control
How Outlet Works in Protected Routes:
  1. Authentication Check: The ProtectedRoute component first checks if the user is authenticated
  2. Conditional Rendering: If authenticated, the <Outlet /> renders the matching child route
  3. Child Routes: Your dashboard, profile, and settings routes are all children of the ProtectedRoute
  4. One Guard, Many Routes: This approach protects multiple routes with a single authentication check

Without Outlet, you'd have to repeat your authentication logic in every protected component, making your code harder to maintain.

Route Guards and Authentication

Route guards are more advanced bouncers that can perform additional checks. For example, they can check not just if a user is logged in, but also what permissions they have:

RouteGuard.jsx 
import { useLocation, Navigate, Outlet } from 'react-router-dom';

function RequireAuth({ allowedRoles }) {
    const { user } = useAuth(); // Your authentication hook (you'd create this)
    const location = useLocation();
    
    // First check: Is the user logged in?
    if (!user) {
        // If not logged in, redirect to login page
        // Also remember where they were trying to go (to redirect back later)
        return <Navigate to="/login" state={{ from: location }} replace />;
    }
    
    // Second check: Does the user have permission?
    if (!allowedRoles.includes(user.role)) {
        // If the user doesn't have the right role (e.g., not an admin)
        return <Navigate to="/unauthorized" replace />;
    }
    
    // If they pass all checks, show the protected content
    return <Outlet />;
}

// In your routes configuration
<Route element={<RequireAuth allowedRoles={['admin']} />}>
    <Route path="/admin" element={<AdminPanel />} />
</Route>

Router Hooks and Utilities

React Router provides special functions called "hooks" that let you access routing information and navigate from anywhere in your components. Think of these hooks as your navigation toolkit:

useNavigate

Programmatic navigation between routes

useLocation

Access current URL information and state

useParams

Extract dynamic parameters from the URL

useSearchParams

Read and modify query string parameters

useNavigate

The useNavigate hook gives you a navigation function you can call from anywhere. Think of it as a "go to page" remote control that lets you change pages programmatically:

Navigation Commands 
import { useNavigate } from 'react-router-dom';

function NavigationExample() {
    // Get the navigate function
    const navigate = useNavigate();
    
    return (
        <div className="navigation-examples">
            <button onClick={() => navigate('/dashboard')}>
                Go to Dashboard
            </button>
            
            {/* Replace current history entry (can't go back) */}
            <button onClick={() => navigate('/profile', { replace: true })}>
                Go to Profile
            </button>
            
            {/* Navigation with browser history */}
            <button onClick={() => navigate(-1)}>Go Back</button>
            <button onClick={() => navigate(1)}>Go Forward</button>
            
            {/* Pass state data to the destination */}
            <button onClick={() => 
                navigate('/details', { state: { from: 'navigation example' } })
            }>
                Go to Details with State
            </button>
        </div>
    );
}

useLocation

The useLocation hook tells you information about the current page URL and any data that was passed during navigation. It's like a GPS that can tell you exactly where you are in your app and how you got there:

Location Commands 
import { useLocation } from 'react-router-dom';

function CurrentRoute() {
    // useLocation gives us details about the current URL
    const location = useLocation();
    
    return (
        <div className="location-info p-4 border rounded">
            <h2>Current Location Information</h2>
            
            {/* URL path information */}
            <p>Current path: {location.pathname}</p>     {/* e.g., "/products/123" */}
            <p>Search params: {location.search}</p>     {/* e.g., "?color=red" */}
            <p>Hash: {location.hash}</p>                {/* e.g., "#section1" */}
            
            {/* Access the invisible state data passed during navigation */}
            {location.state && (
                <div className="state-info mt-4 bg-blue-50 p-3 rounded">
                    <h3>Navigation State Data</h3>
                    <p>Navigated from: {location.state.from}</p>
                    {location.state.timestamp && (
                        <p>Timestamp: {location.state.timestamp}</p>
                    )}
                </div>
            )}
        </div>
    );
}

Tip: useLocation is especially useful for analytics tracking, breadcrumbs, or when you need to respond to URL changes in your app.

useSearchParams

The useSearchParams hook handles query string parameters in your URL (the ?key=value&part=2 part). It lets you read and update these parameters easily:

Search Params Commands 
import { useSearchParams } from 'react-router-dom';

function ProductFilters() {
    // Get and set search parameters from the URL
    const [searchParams, setSearchParams] = useSearchParams();
    
    // Read current filter values from URL
    const category = searchParams.get('category') || 'all';
    const minPrice = searchParams.get('minPrice') || '';
    const maxPrice = searchParams.get('maxPrice') || '';
    
    // Function to update filters in the URL
    const updateFilters = (newCategory, newMinPrice, newMaxPrice) => {
        const params = new URLSearchParams();
        
        if (newCategory && newCategory !== 'all') {
            params.set('category', newCategory);
        }
        
        if (newMinPrice) {
            params.set('minPrice', newMinPrice);
        }
        
        if (newMaxPrice) {
            params.set('maxPrice', newMaxPrice);
        }
        
        // Update URL with new parameters
        setSearchParams(params);
    };
    
    return (
        <div className="product-filters">
            <h2>Product Filters</h2>
            
            <div className="filter-controls">
                <select 
                    value={category}
                    onChange={(e) => updateFilters(e.target.value, minPrice, maxPrice)}
                >
                    <option value="all">All Categories</option>
                    <option value="electronics">Electronics</option>
                    <option value="clothing">Clothing</option>
                </select>
                
                <div className="price-range">
                    <input 
                        type="number" 
                        placeholder="Min Price" 
                        value={minPrice}
                        onChange={(e) => updateFilters(category, e.target.value, maxPrice)}
                    />
                    <input 
                        type="number" 
                        placeholder="Max Price" 
                        value={maxPrice}
                        onChange={(e) => updateFilters(category, minPrice, e.target.value)}
                    />
                </div>
            </div>
            
            <p>Current URL parameters: {window.location.search}</p>
        </div>
    );
}

Route-based Code Splitting

As your app grows, loading everything at once can slow it down. Code splitting lets you load just the code needed for the current page. It's like downloading only the book chapters you're reading right now, rather than the whole book:

AppWithCodeSplitting.jsx 
import { Suspense, lazy } from 'react';
import { BrowserRouter, Routes, Route } from 'react-router-dom';

// Instead of importing components normally:
// import Home from './pages/Home';
// import About from './pages/About';

// Lazy load components - only download them when needed
const Home = lazy(() => import('./pages/Home'));       // Will load when visiting home page
const About = lazy(() => import('./pages/About'));     // Will load when visiting about page
const Dashboard = lazy(() => import('./pages/Dashboard')); // Will load when visiting dashboard

function App() {
    return (
        <BrowserRouter>
            {/* The Suspense component shows a fallback while code is loading */}
            <Suspense fallback={
                <div className="flex justify-center items-center h-screen">
                    <div className="animate-spin rounded-full h-12 w-12 border-t-2 border-b-2 border-purple-500"></div>
                    <span className="ml-3 text-purple-600">Loading...</span>
                </div>
            }>
                <Routes>
                    <Route path="/" element={<Home />} />
                    <Route path="/about" element={<About />} />
                    <Route path="/dashboard" element={<Dashboard />} />
                </Routes>
            </Suspense>
        </BrowserRouter>
    );
}

Code splitting can dramatically improve initial load time by reducing the size of your JavaScript bundle. Users only download what they need when they need it.

Navigation State Management

Sometimes you need to pass information between pages without showing it in the URL. For example, you might want to remember where a user came from or what they were doing. Navigation state lets you attach invisible data to navigation actions:

NavigationState.jsx 
// Passing state during navigation (data that travels with the user)
import { Link, useNavigate } from 'react-router-dom';

function ProductList() {
    const navigate = useNavigate();
    
    return (
        <div className="product-list">
            {/* Method 1: Pass state via Link component */}
            <Link 
                to="/product/1" 
                state={{ 
                    fromList: true,
                    viewMethod: "link" 
                }}
                className="text-blue-500 hover:underline"
            >
                Product 1
            </Link>
            
            {/* Method 2: Pass state programmatically with navigate */}
            <button 
                onClick={() => navigate('/product/2', { 
                    state: { 
                        fromList: true,           // Remember where user came from
                        viewedAt: new Date(),     // Remember when they viewed it
                        viewMethod: "button"      // How they navigated
                    } 
                })}
                className="bg-blue-500 text-white px-4 py-2 rounded ml-4"
            >
                Product 2
            </button>
        </div>
    );
}

// Retrieving state in the destination component
import { useLocation } from 'react-router-dom';

function ProductDetail() {
    const location = useLocation();
    
    // Access the state passed during navigation
    // Use empty object as fallback if no state exists
    const { fromList, viewedAt, viewMethod } = location.state || {};
    
    return (
        <div className="product-detail p-4 border rounded-lg bg-white">
            <h1 className="text-xl font-bold mb-4">Product Detail Page</h1>
            
            {/* Conditional rendering based on navigation state */}
            {fromList && (
                <div className="bg-blue-50 p-3 rounded-lg mb-3">
                    <p>You came from the product list</p>
                    {viewMethod && <p>Navigation method: {viewMethod}</p>}
                </div>
            )}
            
            {viewedAt && (
                <p className="text-gray-500 text-sm">
                    Viewed at: {viewedAt.toString()}
                </p>
            )}
        </div>
    );
}

When to use navigation state: Navigation state is perfect for passing temporary data between routes that shouldn't be in the URL. For persistent data, consider React Context or state management libraries like Redux instead.
