Fetch vs Axios: Key Differences

  • F Fetch: Built into modern browsers, no installation needed
  • A Axios: Automatic JSON transformation, request cancellation, better error handling
  • A Simpler API and consistent cross-browser behavior
  • F Requires more code for error handling (doesn't reject on HTTP error status)

useEffect for Data Fetching

The useEffect hook is perfect for handling data fetching in functional components. It runs after the component renders, allowing you to perform side effects like API calls.

useEffect for API Calls 
import React, { useState, useEffect } from 'react';

function PostDetail({ postId }) {
  const [post, setPost] = useState(null);
  
  useEffect(() => {
    // This effect runs when the component mounts
    // and whenever postId changes
    
    const fetchPost = async () => {
      try {
        const response = await fetch(`https://api.example.com/posts/${postId}`);
        const data = await response.json();
        setPost(data);
      } catch (error) {
        console.error('Error fetching post:', error);
      }
    };
    
    fetchPost();
    
    // Optional cleanup function
    return () => {
      // Cancel any pending requests or cleanup here
      // (useful for abortable fetch or axios cancellation)
    };
  }, [postId]); // Dependency array - effect runs when postId changes
  
  // Rest of component...
}

Important useEffect Notes:

  • 1 Always include dependencies (like postId) that your effect uses
  • 2 Empty dependency array [] means "run once after initial render"
  • 3 No dependency array means "run after every render" (rarely what you want for API calls)
  • 4 Consider using cleanup functions to prevent memory leaks and cancel pending requests

Loading States and Error Handling

Always account for the three main states of API calls: loading, success, and error. Properly handling these states creates a better user experience and more robust applications.

Loading State

Show spinners, skeletons, or loading indicators to inform users that data is being fetched

Success State

Display the fetched data in a user-friendly format once the request completes successfully

Error State

Show meaningful error messages and recovery options when requests fail

Advantages of Async/Await

  • Cleaner, more readable code than promise chains
  • Makes error handling more straightforward with try/catch
  • Easier to debug with standard JavaScript error stacks
  • Allows using regular JavaScript control flow (if/else, loops)

Async/Await Tips

  • Always use try/catch blocks to handle errors
  • Remember that await only works inside async functions
  • Consider using Promise.all() for running multiple fetches in parallel
  • Don't forget about cleanup/cancellation for component unmounting
  • Use AbortController to cancel fetch requests when component unmounts

Advanced Data Fetching

Caching Strategies

Caching fetched data can significantly improve your app's performance and reduce unnecessary network requests.

function useCachedData(url) {
  const cache = useRef({});
  const [data, setData] = useState(null);
  const [isLoading, setIsLoading] = useState(false);
  
  useEffect(() => {
    const fetchData = async () => {
      // Check if we already have cached data
      if (cache.current[url]) {
        setData(cache.current[url]);
        return;
      }
      
      setIsLoading(true);
      try {
        const response = await fetch(url);
        const result = await response.json();
        
        // Store in cache and update state
        cache.current[url] = result;
        setData(result);
      } catch (error) {
        console.error('Fetch error:', error);
      } finally {
        setIsLoading(false);
      }
    };
    
    fetchData();
  }, [url]);
  
  return { data, isLoading };
}

For more advanced caching, consider using libraries like SWR, React Query, or Apollo Client.

Optimistic Updates

Update the UI immediately as if an operation succeeded, then revert if it fails. This creates a more responsive user experience.

function TodoList() {
  const [todos, setTodos] = useState([]);
  
  const addTodo = async (newTodo) => {
    // 1. Create a unique temporary ID
    const tempId = Date.now();
    
    // 2. Optimistically update UI
    const optimisticTodo = { id: tempId, text: newTodo, completed: false };
    setTodos(prevTodos => [...prevTodos, optimisticTodo]);
    
    try {
      // 3. Make the actual API call
      const response = await fetch('https://api.example.com/todos', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ text: newTodo }),
      });
      
      if (!response.ok) throw new Error('Failed to add todo');
      
      // 4. Update with the real data from server (which may include a real ID)
      const savedTodo = await response.json();
      
      setTodos(prevTodos => prevTodos.map(todo => 
        todo.id === tempId ? savedTodo : todo
      ));
      
    } catch (error) {
      // 5. On failure, roll back the optimistic update
      setTodos(prevTodos => prevTodos.filter(todo => todo.id !== tempId));
      alert('Failed to add todo!');
    }
  };
  
  // Rest of component...
}

Custom Hooks for API Calls

Create reusable custom hooks to encapsulate API logic and share it across components.

// hooks/useApi.js
import { useState, useEffect } from 'react';

export function useFetch(url, options = {}) {
  const [data, setData] = useState(null);
  const [error, setError] = useState(null);
  const [loading, setLoading] = useState(false);
  
  useEffect(() => {
    let isMounted = true;
    
    const fetchData = async () => {
      setLoading(true);
      
      try {
        const response = await fetch(url, options);
        
        if (!response.ok) {
          throw new Error(`API error: ${response.status}`);
        }
        
        const result = await response.json();
        
        if (isMounted) {
          setData(result);
          setError(null);
        }
      } catch (err) {
        if (isMounted) {
          setError(err.message || 'An error occurred');
          setData(null);
        }
      } finally {
        if (isMounted) {
          setLoading(false);
        }
      }
    };
    
    fetchData();
    
    return () => {
      isMounted = false;
    };
  }, [url, JSON.stringify(options)]);
  
  return { data, error, loading };
}

Benefits of Custom API Hooks

  • Reusability: Use the same data fetching logic across multiple components
  • Separation of concerns: Isolate API logic from UI components
  • Easier testing: Test data fetching logic separately from components
  • Consistent error handling: Manage errors consistently across your application
  • Easier refactoring: Change the data fetching implementation without modifying components
  • DRY principle: Don't Repeat Yourself by centralizing API logic

Data Normalization

Normalize nested API data to make it easier to update and maintain in your app's state.

// Instead of keeping nested data structures
// Original API Response:
/* 
{
  id: 1,
  title: "Post 1",
  author: {
    id: 101,
    name: "Alice",
    posts: [1, 2, 3]
  },
  comments: [
    { id: 201, text: "Great post!", author: { id: 102, name: "Bob" } },
    { id: 202, text: "Thanks!", author: { id: 101, name: "Alice" } }
  ]
}
*/

// Normalized data structure:
const normalizedState = {
  posts: {
    1: { id: 1, title: "Post 1", author: 101, comments: [201, 202] }
  },
  users: {
    101: { id: 101, name: "Alice", posts: [1, 2, 3] },
    102: { id: 102, name: "Bob", posts: [] }
  },
  comments: {
    201: { id: 201, text: "Great post!", author: 102 },
    202: { id: 202, text: "Thanks!", author: 101 }
  }
};

// Now updating a user's name is easy:
normalizedState.users[101].name = "Alice Smith";

Libraries like Redux recommend this approach. For small apps, it might be overkill, but it's incredibly useful as your app grows.

Key Takeaways

API Integration Basics

  • Use Fetch API (built-in) or Axios (more features)
  • Fetch data in useEffect with proper dependency arrays
  • Always handle loading, success, and error states
  • Use async/await for cleaner asynchronous code

Advanced Techniques

  • Create custom hooks to reuse API logic across components
  • Implement data caching for better performance
  • Use optimistic updates for a more responsive UX
  • Consider specialized libraries for complex data requirements

Next Steps in Your React API Journey

For small to medium applications:

  • Custom hooks with Fetch API or Axios
  • Simple local caching techniques
  • Local state management with useState/useReducer

For larger, data-intensive applications:

  • Explore React Query or SWR
  • Consider RTK Query if already using Redux
  • Try Apollo Client for GraphQL APIs

Common API Integration Patterns

Request-Response

The simplest pattern - send a request and handle the response.

Best for: Query operations, data retrieval, simple form submissions.

Optimistic Updates

Update UI immediately, then confirm with the server.

Best for: Interactive UIs, social features, quick feedback loops.

Real-time Updates

Establish persistent connections for immediate data pushes.

Best for: Chat apps, live feeds, collaborative features.
