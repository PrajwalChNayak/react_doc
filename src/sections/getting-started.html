Getting Started with React

Learn how to set up a React project, understand the library's history and scope, and run your first React application.

What is React?

React is a JavaScript library for building user interfaces, particularly single-page applications. It allows developers to create large web applications that can change data without reloading the page. The main focus of React is on developing components, which are reusable UI elements for your application.

React embraces the concept of unidirectional data flow and a virtual DOM, making it efficient and faster than traditional DOM manipulation. Its component-based architecture encourages the creation of reusable UI components that present data that changes over time.

Key Features of React:

  • Virtual DOM for efficient rendering
  • Component-based architecture
  • JSX syntax for HTML in JavaScript
  • One-way data binding
  • Focus on UI components

Why Use React?

React has become one of the most popular front-end libraries for good reasons. It offers numerous benefits that make development faster, more efficient, and more maintainable.

Reusable Components

Build encapsulated components that manage their own state, then compose them to make complex UIs. Component-based architecture promotes reuse, testing, and separation of concerns.

Developer Experience

React's declarative approach makes your code more predictable and easier to debug. The React ecosystem includes excellent developer tools and extensive documentation.

Performance

React's Virtual DOM and efficient diffing algorithm minimize expensive DOM operations, delivering high performance even for complex UIs with frequently changing data.

Strong Community & Ecosystem

React has a vast ecosystem of libraries, tools, and extensions. It's backed by Meta (Facebook) and used by companies like Netflix, Airbnb, and Instagram.

Seamless Transitions

React can be gradually adopted in existing projects or used to build entire applications from scratch. It works well with other libraries and frameworks.

Mobile Development

Learn once, write anywhere: React Native lets you use the same component principles to build native mobile apps for iOS and Android.

Understanding Virtual DOM

One of React's most important features is the Virtual DOM (Document Object Model). This is a key concept that makes React applications fast and efficient, especially when handling frequent UI updates.

How Virtual DOM Works

What is DOM?

The Document Object Model (DOM) is a programming interface for web documents. It represents the page so that programs can change the document structure, style, and content.

Problem with Direct DOM Manipulation

Directly manipulating the DOM is slow. When updating the DOM, browsers need to recalculate CSS, perform layout, and repaint, which can be inefficient for complex applications.

Virtual DOM Solution

React creates a lightweight copy of the real DOM in memory (Virtual DOM). When state changes, React first updates this Virtual DOM rather than the browser's DOM directly.

Reconciliation Process

React compares the updated Virtual DOM with a pre-update snapshot, identifies the exact changes (diffing), and then updates only those specific parts in the real DOM.

Virtual DOM Process:
  1. 1
    State Changes

    When data changes in a React component

  2. 2
    Virtual DOM Update

    React creates a new Virtual DOM tree with the updated state

  3. 3
    Diffing

    React compares new Virtual DOM with previous version

  4. 4
    Batch Update

    Only the identified differences are applied to the real DOM

Benefits of Virtual DOM:
  • Improved performance for complex UIs
  • Simplified programming model for developers
  • Cross-platform compatibility
  • Reduced memory consumption
  • Better developer experience through predictable updates

The best framework depends on your project needs, team expertise, and specific requirements:

Arrow function:

Extract values from arrays or properties from objects into distinct variables.

Array destructuring:

Spread operator:

// Combine arrays

Rest parameter:

function sum(...numbers) {

Understanding Immutability

React relies heavily on immutable data patterns. This means never directly modifying state objects, but instead creating new copies with changes:

❌ Incorrect (Mutable)
// Directly modifying state
const handleClick = () => {
  user.name = 'New Name';
  user.age += 1;
  setUser(user); // Wrong! Same reference
};
✅ Correct (Immutable)
// Creating new state object
const handleClick = () => {
  setUser({
    ...user,
    name: 'New Name',
    age: user.age + 1
  }); // Correct! New reference
};

History of React

React was first created by Jordan Walke, a software engineer at Facebook, around 2011. It was initially used for Facebook's newsfeed. The first deployment of React was on Facebook's newsfeed in 2011, and later for Instagram's web app in 2012.

Facebook publicly released React as an open-source project at the JSConf US in May 2013. React Native, which enables native Android, iOS, and UWP development with React, was announced at Facebook's React.js Conference in February 2015 and open-sourced in March 2015.

Who Manages React?

React is maintained by Meta (formerly Facebook) and a community of individual developers and companies. The React team at Meta leads the development of the core library, while the broader React community contributes to the ecosystem through plugins, tools, and libraries.

While Meta employs the primary React development team, the project remains open-source, allowing contributions from developers worldwide. React's development is guided by a set of core principles, including component-based architecture, declarative programming, and the virtual DOM concept.

React Management Structure:

  • Meta React Core Team - Primary development and roadmap
  • Open-source contributors - Bug fixes and feature contributions
  • Community maintainers - Ecosystem packages and tools

Scope of React

React's primary purpose is to build user interfaces - what users see and interact with in an application. Unlike full frameworks like Angular or Vue, React focuses solely on the view layer. This specialization is intentional, allowing React to excel at UI rendering while leaving other concerns to complementary libraries.

Because React is focused on UI components, it's often combined with other libraries to create complete applications. For example, React doesn't include built-in routing or state management, so developers typically use React Router and libraries like Redux or Zustand for these features.

What React Does:

  • UI component rendering
  • Virtual DOM handling
  • Component lifecycle management
  • JSX transpilation

What React Doesn't Do:

  • Routing (use React Router)
  • Global state management (use Redux, Zustand)
  • API calls (use Fetch, Axios)
  • Form validation (use Formik, React Hook Form)

React Concepts: Beginner to Advanced

Beginner Concepts

  • Components

    Building blocks of React applications. Functional components are now preferred over class components.

  • JSX

    JavaScript syntax extension that looks similar to HTML but allows you to write HTML-like code in JavaScript.

  • Props

    How data is passed from parent to child components, similar to HTML attributes.

  • State

    Data that changes over time in a component, managed with useState hook in functional components.

Intermediate Concepts

  • Lifecycle & Effects

    Managing component lifecycles with useEffect hook for side effects like data fetching.

  • Context API

    Sharing data across component trees without passing props down manually through each level.

  • React Router

    Navigation and routing in single-page applications to render different components for different URLs.

  • Forms & Controlled Components

    Managing form inputs and validation in React applications.

Advanced Concepts

  • Performance Optimization

    Using React.memo, useMemo, useCallback hooks to optimize rendering performance.

  • Custom Hooks

    Creating reusable stateful logic that can be shared between components.

  • Render Props & HOCs

    Patterns for code reuse and component composition beyond simple component hierarchies.

  • Server Components

    React components that run on the server and can access server data directly (React 18+).

  • Concurrent Mode & Suspense

    New features for improving UI responsiveness with asynchronous rendering.

Installing Node.js and npm/yarn

Before you can start working with React, you'll need to set up Node.js and a package manager like npm or Yarn on your development machine. These tools are essential for creating, managing, and building React applications.

What is Node.js and npm?

Node.js

Node.js is a JavaScript runtime environment that allows you to run JavaScript on your computer outside of a web browser. React development tools and build processes are built on Node.js.

npm (Node Package Manager)

npm is Node.js's default package manager that comes bundled with Node.js installation. It allows you to install and manage JavaScript packages and dependencies for your project.

Installation Options

Option 1: Direct Download (Recommended for Beginners)
Step 1: Download

Visit the official Node.js website and download the LTS (Long Term Support) version, which is recommended for most users.

Step 2: Install

Run the downloaded installer and follow the installation wizard's instructions. The default settings are suitable for most users.

Step 3: Verify

Open your terminal or command prompt and run these commands to verify installation:

node --version
npm --version
Option 2: Using Package Managers
macOS (using Homebrew)
brew update
brew install node

If you don't have Homebrew installed, install it first with:

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
Windows (using Chocolatey)
choco install nodejs-lts

If you don't have Chocolatey installed, follow the instructions at chocolatey.org/install

Linux (Ubuntu/Debian)
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs
Installing Yarn (Alternative to npm)

Yarn is an alternative package manager to npm with some performance and workflow advantages. It's optional but popular among React developers.

Install Yarn with npm
npm install --global yarn
Verify Yarn installation
yarn --version

Key Files and Directories
package.json

Contains project metadata, dependencies, and scripts that can be run (like start, build, test)

public/

Contains static assets that won't be processed by the build tools. The index.html here is the template for your app

src/

The main directory where you'll write your React code

src/index.js (or main.jsx)

The entry point that renders your React app into the DOM

src/App.js (or App.jsx)

The root component of your application

Recommended Structure for Beginners

For those just starting with React, we recommend a simple structure to avoid complexity:

src/
components/ - Reusable UI components
pages/ - Components representing entire pages
hooks/ - Custom React hooks
assets/ - Images, fonts, etc.
utils/ - Utility functions
App.jsx - Root component
main.jsx - Entry point

Start with a simpler structure and refactor as your application grows. The best structure is one that makes sense to you and your team and helps you work efficiently.
