Understanding React Hooks

Hooks are a revolutionary addition to React that allow you to use state and other React features without writing a class. Learn how hooks simplify your components and make code reuse easier.

What are Hooks and Why They Exist

Hooks are functions that let you "hook into" React state and lifecycle features from function components. They were introduced in React 16.8 as a way to use React features without writing classes.

Why Hooks Were Created

  • Reuse stateful logic between components without complex patterns like render props or higher-order components
  • Split complex components into smaller functions based on related pieces, rather than forcing a split based on lifecycle methods
  • Use React features without classes, avoiding the confusion of this binding, understanding JavaScript classes, and making components more friendly to optimization
  • Simplify complex components that become unwieldy with lifecycle methods that contain unrelated logic
Before Hooks (Class Component) 
class Counter extends React.Component {
  constructor(props) {
    super(props);
    this.state = { count: 0 };
    this.handleClick = this.handleClick.bind(this);
  }
  
  handleClick() {
    this.setState({ count: this.state.count + 1 });
  }
  
  render() {
    return (
      <div>
        <p>You clicked {this.state.count} times</p>
        <button onClick={this.handleClick}>Click me</button>
      </div>
    );
  }
}
After Hooks (Function Component) 
function Counter() {
  const [count, setCount] = useState(0);
  
  return (
    <div>
      <p>You clicked {count} times</p>
      <button onClick={() => setCount(count + 1)}>
        Click me
      </button>
    </div>
  );
}

As you can see, Hooks make your code cleaner and more concise. No more complex class syntax, binding, or this keyword confusion.

Rules of Hooks

Hooks come with two essential rules that you must follow for them to work correctly:

1. Only Call Hooks at the Top Level

Don't call Hooks inside loops, conditions, or nested functions. React relies on the order in which Hooks are called to associate hook state with the component.

❌ Don't do this: 
function Counter() {
  // Conditionally using hooks is bad!
  if (condition) {
    const [count, setCount] = useState(0);
  }
  // ...
}
✅ Do this instead: 
function Counter() {
  const [count, setCount] = useState(0);
  
  // Use the state conditionally
  if (condition) {
    // Do something with count
  }
  // ...
}

2. Only Call Hooks from React Functions

Call Hooks only from React function components or from custom Hooks. Don't call them from regular JavaScript functions or class components.

✅ Places where hooks work:
  • React function components
  • Custom Hooks (functions starting with "use")
❌ Places where hooks don't work:
  • Class components
  • Regular JavaScript functions
  • Event handlers (outside component body)

Tip: Use ESLint Plugin

The eslint-plugin-react-hooks package is officially provided by the React team and helps enforce these rules automatically in your code.

Built-in vs Custom Hooks

React provides several built-in Hooks that cover common use cases. You can also create your own custom Hooks to extract and reuse stateful logic across components.

Common Built-in Hooks

State

useState

Adds state to your function component. Returns a stateful value and a function to update it. 

const [state, setState] = useState(initialState);
Lifecycle

useEffect

Performs side effects in function components, similar to componentDidMount, componentDidUpdate, and componentWillUnmount. 

useEffect(() => {
  // Effect code
  return () => {
    // Cleanup code
  };
}, [dependencies]);
Context

useContext

Accepts a context object and returns the current context value, enabling the use of React Context without nesting. 

const value = useContext(MyContext);
State

useReducer

An alternative to useState for more complex state logic, especially when state transitions depend on previous state. 

const [state, dispatch] = useReducer(
  reducer, initialState
);

useCallback

Returns a memoized callback function that only changes if dependencies change.

useMemo

Returns a memoized value that only recomputes when dependencies change.

useRef

Returns a mutable ref object that persists for the lifetime of the component.

Custom Hooks

Custom Hooks let you extract component logic into reusable functions. A custom Hook is a JavaScript function whose name starts with "use" and that may call other Hooks.

Example Custom Hook 
// Custom Hook for form handling
function useFormInput(initialValue) {
  const [value, setValue] = useState(initialValue);
  
  function handleChange(e) {
    setValue(e.target.value);
  }
  
  return {
    value,
    onChange: handleChange
  };
}

// Using the custom Hook in a component
function SignupForm() {
  const name = useFormInput('');
  const email = useFormInput('');
  
  function handleSubmit(e) {
    e.preventDefault();
    alert(`Name: ${name.value}, Email: ${email.value}`);
  }
  
  return (
    <form onSubmit={handleSubmit}>
      <input 
        type="text" 
        placeholder="Name" 
        {...name} 
      />
      <input 
        type="email" 
        placeholder="Email" 
        {...email} 
      />
      <button type="submit">Sign Up</button>
    </form>
  );
}

Benefits of Custom Hooks

  • Code reuse - Extract logic that can be shared between components
  • Cleaner components - Separate concerns and keep component code focused
  • Testability - Custom hooks can be tested independently
  • Composition - Custom hooks can call other hooks, creating powerful compositions

Essential Hooks Quick Reference

A concise guide to React's most important hooks and how to use them effectively.

useState

Adds state to function components 

const [state, setState] = useState(initialValue);
Key features:
  • Functional updates: setState(prev => prev + 1)
  • Object updates: setState({...prev, key: value})
  • Lazy init: useState(() => expensiveCalc())

useEffect

Performs side effects in components 

useEffect(() => {
  // Side effect code
  return () => { /* cleanup */ };
}, [dependencies]);
Dependency patterns:
  • Empty array []: Run once on mount
  • No array: Run on every render
  • With deps [a, b]: Run when deps change

useContext

Consumes context values without nesting 

// Create context
const ThemeContext = createContext('light');

// Provider in parent

  


// Consumer in any child
const theme = useContext(ThemeContext);
Best practices:
  • Create custom context hooks
  • Use specific contexts for different concerns

useReducer

Manages complex state logic 

const [state, dispatch] = useReducer(
  (state, action) => {
    switch (action.type) {
      case 'INCREMENT': return state + 1;
      default: return state;
    }
  }, 
  0 // initial state
);
When to use:
  • Complex state with multiple sub-values
  • When state depends on previous state
  • For more predictable state transitions

useMemo

Memoizes expensive calculations 

const memoizedValue = useMemo(() => {
  return computeExpensiveValue(a, b);
}, [a, b]);
Use cases:
  • Expensive calculations
  • Preserving referential equality for objects/arrays
  • Preventing unnecessary re-renders

useCallback

Memoizes callback functions 

const memoizedCallback = useCallback(
  () => {
    doSomething(a, b);
  },
  [a, b]
);
Best for:
  • Functions passed to optimized child components
  • Used with React.memo() for performance
  • Preventing unnecessary effect triggers

Common Hook Patterns

Data Fetching 
useEffect(() => {
  let isMounted = true;
  const fetchData = async () => {
    setLoading(true);
    try {
      const data = await api.get();
      if (isMounted) setData(data);
    } catch (err) {
      if (isMounted) setError(err);
    } finally {
      if (isMounted) setLoading(false);
    }
  };
  fetchData();
  return () => { isMounted = false };
}, []);
Custom Hooks 
function useLocalStorage(key, initialValue) {
  const [value, setValue] = useState(() => {
    const stored = localStorage.getItem(key);
    return stored ? JSON.parse(stored) : initialValue;
  });

  useEffect(() => {
    localStorage.setItem(key, JSON.stringify(value));
  }, [key, value]);

  return [value, setValue];
}

Hook Tips & Gotchas

  • Only call hooks at the top level of your components
  • Don't call hooks inside loops, conditions, or nested functions
  • Custom hooks should start with "use" (e.g., useFormInput)
  • The dependency array is crucial for useEffect, useMemo, and useCallback
  • Always include all values referenced inside the hook from your component scope
  • Use the exhaustive-deps ESLint rule to catch dependency issues
  • Missing dependencies can cause stale closures
  • Object and array dependencies cause re-renders unless memoized
  • setState is asynchronous; don't rely on state updates immediately
  • Multiple state updates in the same event handler are batched
  • Always return cleanup functions in useEffect when working with subscriptions
  • Use the functional update form for state that depends on previous state

Dependency Arrays Simplified

The dependency array controls when your effects run. Think of it like telling React "only run this effect when these values change":

Empty Array []
useEffect(() => { ... }, [])

Runs once after mount

No Array
useEffect(() => { ... })

Runs after every render

With Dependencies
useEffect(() => { ... }, [count, name])

Runs when dependencies change

Common Beginner Mistakes:

  • Empty dependency array when the effect uses props or state
  • Missing dependencies leading to stale closures and bugs
  • Object/array dependencies causing infinite re-render loops

Cleanup Functions Made Simple

Cleanup functions prevent memory leaks by removing event listeners, subscriptions, or timers when your component unmounts:

Without Cleanup (Bad) 

useEffect(() => {
  // 🚨 This creates a new listener each render
  // but never removes old ones!
  window.addEventListener('resize', handleResize);
}, []);

Memory leak: listeners accumulate

With Cleanup (Good) 

useEffect(() => {
  // ✅ Add listener
  window.addEventListener('resize', handleResize);
  
  // ✅ Clean up by removing listener
  return () => {
    window.removeEventListener('resize', handleResize);
  };
}, []);

Custom Hooks: Code Superpowers

Custom hooks let you extract and reuse logic across components, keeping your code DRY and focused:

Creating a Custom Hook 

// 1. Name starts with "use"
function useWindowSize() {
  // 2. Use other hooks inside
  const [size, setSize] = useState({
    width: window.innerWidth,
    height: window.innerHeight
  });
  
  useEffect(() => {
    const handleResize = () => {
      setSize({
        width: window.innerWidth,
        height: window.innerHeight
      });
    };
    
    window.addEventListener('resize', handleResize);
    return () => window.removeEventListener('resize', handleResize);
  }, []);
  
  // 3. Return values
  return size;
}

Real-world Example: useProductSearch 

// A practical hook that combines multiple hooks
function useProductSearch(query, category) {
  // Uses useState internally  
  const [results, setResults] = useState([]);
  const [isLoading, setIsLoading] = useState(false);
  const [error, setError] = useState(null);
  
  // Uses useMemo to construct the search URL
  const searchUrl = useMemo(() => {
    return `/api/products?q=${query}${category ? `&category=${category}` : ''}`;
  }, [query, category]);
  
  // Uses useCallback to memoize the search function
  const search = useCallback(async () => {
    if (!query) return;
    
    try {
      setIsLoading(true);
      setError(null);
      const response = await fetch(searchUrl);
      const data = await response.json();
      setResults(data);
    } catch (err) {
      setError(err.message);
      setResults([]);
    } finally {
      setIsLoading(false);
    }
  }, [searchUrl]);
  
  // Uses useEffect to trigger the search
  useEffect(() => {
    search();
  }, [search]);
  
  return { results, isLoading, error, refresh: search };
}
