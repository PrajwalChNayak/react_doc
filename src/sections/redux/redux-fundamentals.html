Redux Fundamentals

A complete beginner's guide to understanding Redux

What is Redux and Why Use It?

üè™ Think of Redux like a Bank

Imagine your application's data (state) is like money, and Redux is like a bank. Instead of keeping your money under your mattress (in individual components), you store it all safely in one place - the bank (Redux store). When you need money, you go to the bank with a withdrawal slip (action), the teller (reducer) processes it, and gives you what you need.

Problems Redux Solves

  • Prop Drilling: No more passing data through 10+ components
  • Shared State: Multiple components can access the same data
  • Predictability: State changes are predictable and traceable
  • Debugging: Time-travel debugging with DevTools

When NOT to Use Redux

  • Simple apps with minimal state
  • Static websites or landing pages
  • Prototypes or learning projects
  • Apps with only local component state

‚ú® When Redux Shines

Use Redux when you have complex applications with lots of state that needs to be shared between many components, especially when state changes frequently and you need to track those changes for debugging.

The Three Core Principles

1. Single Source of Truth

All your app's state lives in one place - the Redux store.

Analogy: Like having one master filing cabinet for all your important documents instead of scattered papers everywhere.

2. State is Read-Only

You can't directly change the state. You must send an "action" to describe what happened.

Analogy: Like a vending machine - you can't reach in and grab a snack. You must insert coins and press a button.

3. Pure Functions

State changes are made by pure functions called "reducers" that are predictable and testable.

Analogy: Like a recipe - same ingredients always produce the same result.

Essential Redux Building Blocks

The Store

The central repository for all your app's state

üè™ Simple Explanation: The store is like a big container that holds all the data your app needs. It's the only place where your app's state lives. Think of it as your app's memory - everything it needs to remember is stored here.

What it holds:
  • ‚Ä¢ User information
  • ‚Ä¢ App settings
  • ‚Ä¢ UI state (loading, errors)
  • ‚Ä¢ Any shared data
What it provides:
  • ‚Ä¢ getState() - read current state
  • ‚Ä¢ dispatch() - send actions
  • ‚Ä¢ subscribe() - listen for changes
Key features:
  • ‚Ä¢ Single source of truth
  • ‚Ä¢ Immutable updates
  • ‚Ä¢ Predictable changes
  • ‚Ä¢ DevTools integration

Actions

Messages that describe what happened in your app

üì® Simple Explanation: Actions are like messages or events. When something happens in your app (user clicks a button, data loads, etc.), you create an action to describe what happened. It's like saying "Hey Redux, the user just added a new item to their cart!"

// Actions are just plain JavaScript objects
const addTodoAction = {
type: 'ADD_TODO', // What happened
payload: { // Extra data
id: 1,
text: 'Learn Redux'
}
}

Reducers

Pure functions that update state based on actions

üçé‚û°Ô∏èü•ß Simple Explanation: A reducer is like a chef following a recipe. Give it the current state (ingredients) and an action (recipe instructions), and it returns a new state (finished dish). The same inputs always produce the same output - no surprises!

// Reducer: (currentState, action) => newState
function todoReducer(state = [], action) {
switch (action.type) {
case 'ADD_TODO':
return [...state, action.payload]; // New array
default:
return state; // No change
}
}

Dispatch

The only way to trigger state changes

üì¨ Simple Explanation: Dispatch is like mailing a letter. When you want to change the state, you "dispatch" (send) an action to the store. It's the postal service that delivers your action to the reducer, which then updates the state.

// Dispatching an action
store.dispatch({
type: 'ADD_TODO',
payload: { id: 1, text: 'Learn Redux' }
});
// This triggers the reducer and updates the state!

Redux Data Flow - Step by Step

üîÑ The Redux Cycle Explained

Think of Redux data flow like ordering food at a restaurant. You (component) tell the waiter (dispatch) what you want (action), the waiter gives it to the kitchen (reducer), the kitchen prepares your meal (new state), and you get your food (component updates).

1

üñ±Ô∏è User Interaction

User clicks a button, submits a form, or triggers any event in your React component.

Example: User clicks "Add Todo" button
2

üì® Dispatch Action

Your component creates and dispatches an action describing what happened.

dispatch({ type: 'ADD_TODO', payload: 'Learn Redux' })
3

‚öôÔ∏è Reducer Processes

Redux calls your reducer function with the current state and the action.

The reducer looks at the action type and decides how to update the state
4

üíæ State Updates

The reducer returns a new state object, and the store updates its state.

New todo item is added to the state array
5

üîÑ Components Re-render

All components connected to the store automatically re-render with the new state.

Todo list component updates to show the new todo item

üéØ Key Points to Remember

  • State changes are predictable - same action always produces same result
  • Data flows in one direction - from store to components
  • Components never directly modify the state
  • All state changes are traceable through actions

Actions

Actions are plain JavaScript objects that describe what happened.

actions.js 
// Action Types
const ADD_TODO = 'ADD_TODO';
const TOGGLE_TODO = 'TOGGLE_TODO';
const SET_FILTER = 'SET_FILTER';

// Action Creators
export const addTodo = (text) => ({
  type: ADD_TODO,
  payload: {
    id: Date.now(),
    text,
    completed: false
  }
});

export const toggleTodo = (id) => ({
  type: TOGGLE_TODO,
  payload: { id }
});

export const setFilter = (filter) => ({
  type: SET_FILTER,
  payload: { filter }
});

Reducers

Reducers specify how the application's state changes in response to actions.

reducers.js 
const initialState = {
  todos: [],
  filter: 'ALL'
};

const todoReducer = (state = initialState, action) => {
  switch (action.type) {
    case 'ADD_TODO':
      return {
        ...state,
        todos: [...state.todos, action.payload]
      };
      
    case 'TOGGLE_TODO':
      return {
        ...state,
        todos: state.todos.map(todo =>
          todo.id === action.payload.id
            ? { ...todo, completed: !todo.completed }
            : todo
        )
      };
      
    case 'SET_FILTER':
      return {
        ...state,
        filter: action.payload.filter
      };
      
    default:
      return state;
  }
};

Store

The store holds the complete state tree of your application.

store.js 
import { createStore } from 'redux';
import todoReducer from './reducers';

const store = createStore(
  todoReducer,
  window.__REDUX_DEVTOOLS_EXTENSION__ && window.__REDUX_DEVTOOLS_EXTENSION__()
);

// Using the store
store.dispatch(addTodo('Learn Redux'));
store.dispatch(toggleTodo(1));

// Subscribe to changes
store.subscribe(() => {
  console.log('State updated:', store.getState());
});

Practical Examples for Beginners

üéØ Learning by Example: The best way to understand Redux is to see it in action! Let's build some simple, real-world examples that demonstrate each concept clearly.

Example 1: Simple Counter (Your First Redux App)

What we're building: A simple counter that can increment, decrement, and reset. Perfect for understanding the Redux flow!

Step 1: Define Actions
counterActions.js 
// Action Types - What can happen to our counter?
const INCREMENT = 'INCREMENT';
const DECREMENT = 'DECREMENT';
const RESET = 'RESET';
const INCREMENT_BY_AMOUNT = 'INCREMENT_BY_AMOUNT';

// Action Creators - Functions that create actions
export const increment = () => ({
  type: INCREMENT
  // No payload needed - just increment by 1
});

export const decrement = () => ({
  type: DECREMENT
  // No payload needed - just decrement by 1
});

export const reset = () => ({
  type: RESET
  // Reset to 0
});

export const incrementByAmount = (amount) => ({
  type: INCREMENT_BY_AMOUNT,
  payload: amount  // How much to increment by
});

// Example usage:
// dispatch(increment());           // Count goes up by 1
// dispatch(decrement());           // Count goes down by 1
// dispatch(incrementByAmount(5));  // Count goes up by 5
// dispatch(reset());               // Count becomes 0
Step 2: Create Reducer
counterReducer.js 
// Initial state - what our counter starts with
const initialState = {
  count: 0
};

// Reducer function - handles all state changes
const counterReducer = (state = initialState, action) => {
  // Use switch statement to handle different action types
  switch (action.type) {
    case 'INCREMENT':
      return {
        ...state,           // Copy all existing state
        count: state.count + 1  // Update only the count
      };
      
    case 'DECREMENT':
      return {
        ...state,
        count: state.count - 1
      };
      
    case 'RESET':
      return {
        ...state,
        count: 0            // Reset to 0
      };
      
    case 'INCREMENT_BY_AMOUNT':
      return {
        ...state,
        count: state.count + action.payload  // Use the payload
      };
      
    default:
      return state;         // Always return current state for unknown actions
  }
};

export default counterReducer;
Step 3: Create Store
store.js 
import { createStore } from 'redux';
import counterReducer from './counterReducer';

// Create the store with our reducer
const store = createStore(
  counterReducer,
  // Enable Redux DevTools Extension (for debugging)
  window.__REDUX_DEVTOOLS_EXTENSION__ && window.__REDUX_DEVTOOLS_EXTENSION__()
);

export default store;
Step 4: Connect to React
Counter.jsx 
import React from 'react';
import { useSelector, useDispatch } from 'react-redux';
import { increment, decrement, reset, incrementByAmount } from './counterActions';

function Counter() {
  // Get data from Redux store
  const count = useSelector(state => state.count);
  
  // Get dispatch function to send actions
  const dispatch = useDispatch();

  return (
    <div style={{ textAlign: 'center', padding: '20px' }}>
      <h1>Counter: {count}</h1>
      
      <div>
        <button onClick={() => dispatch(decrement())}>
          - Decrease
        </button>
        
        <button onClick={() => dispatch(increment())}>
          + Increase
        </button>
        
        <button onClick={() => dispatch(incrementByAmount(5))}>
          +5 Add Five
        </button>
        
        <button onClick={() => dispatch(reset())}>
          üîÑ Reset
        </button>
      </div>
    </div>
  );
}

export default Counter;
üéâ How it Works:
  1. 1. User clicks "+Increase" button
  2. 2. dispatch(increment()) sends action to store
  3. 3. Reducer receives action and returns new state with count + 1
  4. 4. Component automatically re-renders with new count
  5. 5. User sees updated number on screen!

Example 2: Todo List (Multiple Items & Complex State)

What we're building: A todo list where you can add, toggle completion, and delete todos. This shows how to handle arrays and more complex state!

Todo Actions
todoActions.js 
// Action Types
const ADD_TODO = 'ADD_TODO';
const TOGGLE_TODO = 'TOGGLE_TODO';
const DELETE_TODO = 'DELETE_TODO';

// Action Creators
export const addTodo = (text) => ({

  payload: {
    id: Date.now(),        // Simple ID generation
    text: text,
    completed: false
  }
});

export const toggleTodo = (id) => ({
  type: TOGGLE_TODO,
  payload: { id }
});

export const deleteTodo = (id) => ({
  type: DELETE_TODO,
  payload: { id }
});

// Example usage:
// dispatch(addTodo('Learn Redux'));
// dispatch(toggleTodo(1623456789));
// dispatch(deleteTodo(1623456789));
Todo Reducer
todoReducer.js 
const initialState = {
  todos: []  // Start with empty array
};

const todoReducer = (state = initialState, action) => {
  switch (action.type) {
    case 'ADD_TODO':
      return {
        ...state,
        todos: [...state.todos, action.payload]  // Add new todo to end
      };
      
    case 'TOGGLE_TODO':
      return {
        ...state,
        todos: state.todos.map(todo =>
          todo.id === action.payload.id
            ? { ...todo, completed: !todo.completed }  // Toggle this todo
            : todo                                       // Keep others same
        )
      };
      
    case 'DELETE_TODO':
      return {
        ...state,
        todos: state.todos.filter(todo => 
          todo.id !== action.payload.id  // Keep all except deleted one
        )
      };
      
    default:
      return state;
  }
};

export default todoReducer;
Todo Component
TodoApp.jsx 
import React, { useState } from 'react';
import { useSelector, useDispatch } from 'react-redux';
import { addTodo, toggleTodo, deleteTodo } from './todoActions';

function TodoApp() {
  const [inputText, setInputText] = useState('');
  
  // Get todos from Redux store
  const todos = useSelector(state => state.todos);
  const dispatch = useDispatch();

  const handleSubmit = (e) => {
    e.preventDefault();
    if (inputText.trim()) {
      dispatch(addTodo(inputText));  // Add new todo
      setInputText('');              // Clear input
    }
  };

  return (
    <div>
      <h1>My Todo List</h1>
      
      {/* Add new todo form */}
      <form onSubmit={handleSubmit}>
        <input
          type="text"
          value={inputText}
          onChange={(e) => setInputText(e.target.value)}
          placeholder="What needs to be done?"
        />
        <button type="submit">Add Todo</button>
      </form>

      {/* Display todos */}
      <ul>
        {todos.map(todo => (
          <li key={todo.id}>
            <span
              style={{
                textDecoration: todo.completed ? 'line-through' : 'none',
                cursor: 'pointer'
              }}
              onClick={() => dispatch(toggleTodo(todo.id))}
            >
              {todo.text}
            </span>
            
            <button onClick={() => dispatch(deleteTodo(todo.id))}>
              Delete
            </button>
          </li>
        ))}
      </ul>
      
      <p>Total todos: {todos.length}</p>
      <p>Completed: {todos.filter(t => t.completed).length}</p>
    </div>
  );
}

export default TodoApp;
üí° Key Learning Points:
  • ‚Ä¢ Arrays in state: Use spread operator to add items, map() to update, filter() to delete
  • ‚Ä¢ Unique IDs: Each todo needs a unique ID to identify it for updates/deletes
  • ‚Ä¢ Immutability: Never modify the original array, always create new ones
  • ‚Ä¢ Multiple actions: One reducer can handle many different action types

Common Beginner Mistakes to Avoid

‚ùå Don't Do This:
  • ‚Ä¢ Mutating state directly: state.count++
  • ‚Ä¢ Forgetting to return new state
  • ‚Ä¢ Using Redux for everything (even simple local state)
  • ‚Ä¢ Not using action creators
  • ‚Ä¢ Making API calls in reducers
‚úÖ Do This Instead:
  • ‚Ä¢ Always return new objects: {...state, count: state.count + 1}
  • ‚Ä¢ Use spread operator and array methods
  • ‚Ä¢ Keep local state local, global state global
  • ‚Ä¢ Create action creator functions
  • ‚Ä¢ Keep reducers pure and simple
