Redux Fundamentals

A complete beginner's guide to understanding Redux

What is Redux and Why Use It?

🏪 Think of Redux like a Bank

Imagine your application's data (state) is like money, and Redux is like a bank. Instead of keeping your money under your mattress (in individual components), you store it all safely in one place - the bank (Redux store). When you need money, you go to the bank with a withdrawal slip (action), the teller (reducer) processes it, and gives you what you need.

Problems Redux Solves

  • Prop Drilling: No more passing data through 10+ components
  • Shared State: Multiple components can access the same data
  • Predictability: State changes are predictable and traceable
  • Debugging: Time-travel debugging with DevTools

When NOT to Use Redux

  • Simple apps with minimal state
  • Static websites or landing pages
  • Prototypes or learning projects
  • Apps with only local component state

✨ When Redux Shines

Use Redux when you have complex applications with lots of state that needs to be shared between many components, especially when state changes frequently and you need to track those changes for debugging.

The Three Core Principles

1. Single Source of Truth

All your app's state lives in one place - the Redux store.

Analogy: Like having one master filing cabinet for all your important documents instead of scattered papers everywhere.

2. State is Read-Only

You can't directly change the state. You must send an "action" to describe what happened.

Analogy: Like a vending machine - you can't reach in and grab a snack. You must insert coins and press a button.

3. Pure Functions

State changes are made by pure functions called "reducers" that are predictable and testable.

Analogy: Like a recipe - same ingredients always produce the same result.

Essential Redux Building Blocks

The Store

The central repository for all your app's state

🏪 Simple Explanation: The store is like a big container that holds all the data your app needs. It's the only place where your app's state lives. Think of it as your app's memory - everything it needs to remember is stored here.

What it holds:
  • • User information
  • • App settings
  • • UI state (loading, errors)
  • • Any shared data
What it provides:
  • • getState() - read current state
  • • dispatch() - send actions
  • • subscribe() - listen for changes
Key features:
  • • Single source of truth
  • • Immutable updates
  • • Predictable changes
  • • DevTools integration

Actions

Messages that describe what happened in your app

📨 Simple Explanation: Actions are like messages or events. When something happens in your app (user clicks a button, data loads, etc.), you create an action to describe what happened. It's like saying "Hey Redux, the user just added a new item to their cart!"

// Actions are just plain JavaScript objects
const addTodoAction = {
type: 'ADD_TODO', // What happened
payload: { // Extra data
id: 1,
text: 'Learn Redux'
}
}

Reducers

Pure functions that update state based on actions

🍎➡️🥧 Simple Explanation: A reducer is like a chef following a recipe. Give it the current state (ingredients) and an action (recipe instructions), and it returns a new state (finished dish). The same inputs always produce the same output - no surprises!

// Reducer: (currentState, action) => newState
function todoReducer(state = [], action) {
switch (action.type) {
case 'ADD_TODO':
return [...state, action.payload]; // New array
default:
return state; // No change
}
}

Dispatch

The only way to trigger state changes

📬 Simple Explanation: Dispatch is like mailing a letter. When you want to change the state, you "dispatch" (send) an action to the store. It's the postal service that delivers your action to the reducer, which then updates the state.

// Dispatching an action
store.dispatch({
type: 'ADD_TODO',
payload: { id: 1, text: 'Learn Redux' }
});
// This triggers the reducer and updates the state!

Redux Data Flow - Step by Step

🔄 The Redux Cycle Explained

Think of Redux data flow like ordering food at a restaurant. You (component) tell the waiter (dispatch) what you want (action), the waiter gives it to the kitchen (reducer), the kitchen prepares your meal (new state), and you get your food (component updates).

1

🖱️ User Interaction

User clicks a button, submits a form, or triggers any event in your React component.

Example: User clicks "Add Todo" button
2

📨 Dispatch Action

Your component creates and dispatches an action describing what happened.

dispatch({ type: 'ADD_TODO', payload: 'Learn Redux' })
3

⚙️ Reducer Processes

Redux calls your reducer function with the current state and the action.

The reducer looks at the action type and decides how to update the state
4

💾 State Updates

The reducer returns a new state object, and the store updates its state.

New todo item is added to the state array
5

🔄 Components Re-render

All components connected to the store automatically re-render with the new state.

Todo list component updates to show the new todo item

🎯 Key Points to Remember

  • State changes are predictable - same action always produces same result
  • Data flows in one direction - from store to components
  • Components never directly modify the state
  • All state changes are traceable through actions

Actions

Actions are plain JavaScript objects that describe what happened.

actions.js 
// Action Types
const ADD_TODO = 'ADD_TODO';
const TOGGLE_TODO = 'TOGGLE_TODO';
const SET_FILTER = 'SET_FILTER';

// Action Creators
export const addTodo = (text) => ({
  type: ADD_TODO,
  payload: {
    id: Date.now(),
    text,
    completed: false
  }
});

export const toggleTodo = (id) => ({
  type: TOGGLE_TODO,
  payload: { id }
});

export const setFilter = (filter) => ({
  type: SET_FILTER,
  payload: { filter }
});

Reducers

Reducers specify how the application's state changes in response to actions.

reducers.js 
const initialState = {
  todos: [],
  filter: 'ALL'
};

const todoReducer = (state = initialState, action) => {
  switch (action.type) {
    case 'ADD_TODO':
      return {
        ...state,
        todos: [...state.todos, action.payload]
      };
      
    case 'TOGGLE_TODO':
      return {
        ...state,
        todos: state.todos.map(todo =>
          todo.id === action.payload.id
            ? { ...todo, completed: !todo.completed }
            : todo
        )
      };
      
    case 'SET_FILTER':
      return {
        ...state,
        filter: action.payload.filter
      };
      
    default:
      return state;
  }
};

Store

The store holds the complete state tree of your application.

store.js 
import { createStore } from 'redux';
import todoReducer from './reducers';

const store = createStore(
  todoReducer,
  window.__REDUX_DEVTOOLS_EXTENSION__ && window.__REDUX_DEVTOOLS_EXTENSION__()
);

// Using the store
store.dispatch(addTodo('Learn Redux'));
store.dispatch(toggleTodo(1));

// Subscribe to changes
store.subscribe(() => {
  console.log('State updated:', store.getState());
});

Practical Examples for Beginners

🎯 Learning by Example: The best way to understand Redux is to see it in action! Let's build some simple, real-world examples that demonstrate each concept clearly.

Example 1: Simple Counter (Your First Redux App)

What we're building: A simple counter that can increment, decrement, and reset. Perfect for understanding the Redux flow!

Step 1: Define Actions
counterActions.js 
// Action Types - What can happen to our counter?
const INCREMENT = 'INCREMENT';
const DECREMENT = 'DECREMENT';
const RESET = 'RESET';
const INCREMENT_BY_AMOUNT = 'INCREMENT_BY_AMOUNT';

// Action Creators - Functions that create actions
export const increment = () => ({
  type: INCREMENT
  // No payload needed - just increment by 1
});

export const decrement = () => ({
  type: DECREMENT
  // No payload needed - just decrement by 1
});

export const reset = () => ({
  type: RESET
  // Reset to 0
});

export const incrementByAmount = (amount) => ({
  type: INCREMENT_BY_AMOUNT,
  payload: amount  // How much to increment by
});

// Example usage:
// dispatch(increment());           // Count goes up by 1
// dispatch(decrement());           // Count goes down by 1
// dispatch(incrementByAmount(5));  // Count goes up by 5
// dispatch(reset());               // Count becomes 0
Step 2: Create Reducer
counterReducer.js 
// Initial state - what our counter starts with
const initialState = {
  count: 0
};

// Reducer function - handles all state changes
const counterReducer = (state = initialState, action) => {
  // Use switch statement to handle different action types
  switch (action.type) {
    case 'INCREMENT':
      return {
        ...state,           // Copy all existing state
        count: state.count + 1  // Update only the count
      };
      
    case 'DECREMENT':
      return {
        ...state,
        count: state.count - 1
      };
      
    case 'RESET':
      return {
        ...state,
        count: 0            // Reset to 0
      };
      
    case 'INCREMENT_BY_AMOUNT':
      return {
        ...state,
        count: state.count + action.payload  // Use the payload
      };
      
    default:
      return state;         // Always return current state for unknown actions
  }
};

export default counterReducer;
Step 3: Create Store
store.js 
import { createStore } from 'redux';
import counterReducer from './counterReducer';

// Create the store with our reducer
const store = createStore(
  counterReducer,
  // Enable Redux DevTools Extension (for debugging)
  window.__REDUX_DEVTOOLS_EXTENSION__ && window.__REDUX_DEVTOOLS_EXTENSION__()
);

export default store;
Step 4: Connect to React
Counter.jsx 
import React from 'react';
import { useSelector, useDispatch } from 'react-redux';
import { increment, decrement, reset, incrementByAmount } from './counterActions';

function Counter() {
  // Get data from Redux store
  const count = useSelector(state => state.count);
  
  // Get dispatch function to send actions
  const dispatch = useDispatch();

  return (
    <div style={{ textAlign: 'center', padding: '20px' }}>
      <h1>Counter: {count}</h1>
      
      <div>
        <button onClick={() => dispatch(decrement())}>
          - Decrease
        </button>
        
        <button onClick={() => dispatch(increment())}>
          + Increase
        </button>
        
        <button onClick={() => dispatch(incrementByAmount(5))}>
          +5 Add Five
        </button>
        
        <button onClick={() => dispatch(reset())}>
          🔄 Reset
        </button>
      </div>
    </div>
  );
}

export default Counter;
🎉 How it Works:
  1. 1. User clicks "+Increase" button
  2. 2. dispatch(increment()) sends action to store
  3. 3. Reducer receives action and returns new state with count + 1
  4. 4. Component automatically re-renders with new count
  5. 5. User sees updated number on screen!

// dispatch(addTodo('Learn Redux'));

Todo Reducer
todoReducer.js 
const initialState = {
  todos: []  // Start with empty array
};

const todoReducer = (state = initialState, action) => {
  switch (action.type) {
    case 'ADD_TODO':
      return {
        ...state,
        todos: [...state.todos, action.payload]  // Add new todo to end
      };
      
    case 'TOGGLE_TODO':
      return {
        ...state,
        todos: state.todos.map(todo =>
          todo.id === action.payload.id
            ? { ...todo, completed: !todo.completed }  // Toggle this todo
            : todo                                       // Keep others same
        )
      };
      
    case 'DELETE_TODO':
      return {
        ...state,
        todos: state.todos.filter(todo => 
          todo.id !== action.payload.id  // Keep all except deleted one
        )
      };
      
    default:
      return state;
  }
};

export default todoReducer;
Todo Component
TodoApp.jsx 
import React, { useState } from 'react';
import { useSelector, useDispatch } from 'react-redux';
import { addTodo, toggleTodo, deleteTodo } from './todoActions';

function TodoApp() {
  const [inputText, setInputText] = useState('');
  
  // Get todos from Redux store
  const todos = useSelector(state => state.todos);
  const dispatch = useDispatch();

  const handleSubmit = (e) => {
    e.preventDefault();
    if (inputText.trim()) {
      dispatch(addTodo(inputText));  // Add new todo
      setInputText('');              // Clear input
    }
  };

  return (
    <div>
      <h1>My Todo List</h1>
      
      {/* Add new todo form */}
      <form onSubmit={handleSubmit}>
        <input
          type="text"
          value={inputText}
          onChange={(e) => setInputText(e.target.value)}
          placeholder="What needs to be done?"
        />
        <button type="submit">Add Todo</button>
      </form>

      {/* Display todos */}
      <ul>
        {todos.map(todo => (
          <li key={todo.id}>
            <span
              style={{
                textDecoration: todo.completed ? 'line-through' : 'none',
                cursor: 'pointer'
              }}
              onClick={() => dispatch(toggleTodo(todo.id))}
            >
              {todo.text}
            </span>
            
            <button onClick={() => dispatch(deleteTodo(todo.id))}>
              Delete
            </button>
          </li>
        ))}
      </ul>
      
      <p>Total todos: {todos.length}</p>
      <p>Completed: {todos.filter(t => t.completed).length}</p>
    </div>
  );
}

export default TodoApp;
💡 Key Learning Points:
  • • Arrays in state: Use spread operator to add items, map() to update, filter() to delete
  • • Unique IDs: Each todo needs a unique ID to identify it for updates/deletes
  • • Immutability: Never modify the original array, always create new ones
  • • Multiple actions: One reducer can handle many different action types

Common Beginner Mistakes to Avoid

❌ Don't Do This:
  • • Mutating state directly: state.count++
  • • Forgetting to return new state
  • • Using Redux for everything (even simple local state)
  • • Not using action creators
  • • Making API calls in reducers
✅ Do This Instead:
  • • Always return new objects: {...state, count: state.count + 1}
  • • Use spread operator and array methods
  • • Keep local state local, global state global
  • • Create action creator functions
  • • Keep reducers pure and simple

Advanced RTK Features (When You're Ready)

createAsyncThunk - API Calls Made Easy

What it does: Handles async operations (like API calls) and automatically manages loading/success/error states. No more manual action types for async!

userSlice.js 
import { createSlice, createAsyncThunk } from '@reduxjs/toolkit';

// Create async thunk for fetching user data
export const fetchUser = createAsyncThunk(
  'user/fetchUser',                           // Action type prefix
  async (userId) => {                         // Async function
    const response = await fetch(`/api/users/${userId}`);
    return response.json();                   // This becomes action.payload
  }
);

const userSlice = createSlice({
  name: 'user',
  initialState: {
    data: null,
    loading: false,
    error: null
  },
  reducers: {
    clearUser: (state) => {
      state.data = null;
    }
  },
  extraReducers: (builder) => {
    builder
      .addCase(fetchUser.pending, (state) => {
        state.loading = true;                 // Auto-handled loading state
        state.error = null;
      })
      .addCase(fetchUser.fulfilled, (state, action) => {
        state.loading = false;
        state.data = action.payload;          // API response data
      })
      .addCase(fetchUser.rejected, (state, action) => {
        state.loading = false;
        state.error = action.error.message;   // Auto-handled error
      });
  }
});

export const { clearUser } = userSlice.actions;
export default userSlice.reducer;

// Usage in component:
// dispatch(fetchUser(123)); // Automatically handles loading/success/error!

Practical Example: Todo App with RTK

🎯 Let's build something real! Here's how to create a complete todo app using RTK. You'll see how much cleaner and easier it is compared to traditional Redux!

Complete Todo Slice

todoSlice.js 
import { createSlice, createAsyncThunk } from '@reduxjs/toolkit';

// Async thunk for fetching todos from API
export const fetchTodos = createAsyncThunk(
  'todos/fetchTodos',
  async () => {
    const response = await fetch('/api/todos');
    return response.json();
  }
);

// Async thunk for adding todo to server
export const addTodoAsync = createAsyncThunk(
  'todos/addTodo',
  async (todoText) => {
    const response = await fetch('/api/todos', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ text: todoText, completed: false })
    });
    return response.json();
  }
);

const todoSlice = createSlice({
  name: 'todos',
  initialState: {
    items: [],
    status: 'idle', // 'idle' | 'loading' | 'succeeded' | 'failed'
    error: null,
    filter: 'all' // 'all' | 'active' | 'completed'
  },
  reducers: {
    // Synchronous actions
    todoAdded: (state, action) => {
      state.items.push({
        id: Date.now(),
        text: action.payload,
        completed: false
      });
    },
    todoToggled: (state, action) => {
      const todo = state.items.find(todo => todo.id === action.payload);
      if (todo) {
        todo.completed = !todo.completed; // Direct mutation thanks to Immer!
      }
    },
    todoDeleted: (state, action) => {
      state.items = state.items.filter(todo => todo.id !== action.payload);
    },
    allCompleted: (state) => {
      state.items.forEach(todo => {
        todo.completed = true; // Mutate all items easily
      });
    },
    completedCleared: (state) => {
      state.items = state.items.filter(todo => !todo.completed);
    },
    filterChanged: (state, action) => {
      state.filter = action.payload;
    }
  },
  extraReducers: (builder) => {
    builder
      // Handle fetchTodos
      .addCase(fetchTodos.pending, (state) => {
        state.status = 'loading';
      })
      .addCase(fetchTodos.fulfilled, (state, action) => {
        state.status = 'succeeded';
        state.items = action.payload;
      })
      .addCase(fetchTodos.rejected, (state, action) => {
        state.status = 'failed';
        state.error = action.error.message;
      })
      // Handle addTodoAsync
      .addCase(addTodoAsync.fulfilled, (state, action) => {
        state.items.push(action.payload);
      });
  }
});

// Export actions
export const {
  todoAdded,
  todoToggled,
  todoDeleted,
  allCompleted,
  completedCleared,
  filterChanged
} = todoSlice.actions;

// Export selectors
export const selectAllTodos = (state) => state.todos.items;
export const selectTodosByFilter = (state) => {
  const { items, filter } = state.todos;
  switch (filter) {
    case 'active':
      return items.filter(todo => !todo.completed);
    case 'completed':
      return items.filter(todo => todo.completed);
    default:
      return items;
  }
};

export default todoSlice.reducer;
🎉 Look How Clean This Is!
  • • One file handles all todo logic
  • • Direct mutations like todo.completed = !todo.completed
  • • Async actions with automatic loading states
  • • Built-in selectors for filtering
  • • No action types to manually define

Using the Todo Slice in React

TodoApp.jsx 
import React, { useState, useEffect } from 'react';
import { useSelector, useDispatch } from 'react-redux';
import {
  fetchTodos,
  todoAdded,
  todoToggled,
  todoDeleted,
  filterChanged,
  selectTodosByFilter
} from './todoSlice';

function TodoApp() {
  const [inputText, setInputText] = useState('');
  const dispatch = useDispatch();
  
  // Get data from store using selectors
  const todos = useSelector(selectTodosByFilter);
  const filter = useSelector(state => state.todos.filter);
  const status = useSelector(state => state.todos.status);
  const error = useSelector(state => state.todos.error);

  // Load todos on component mount
  useEffect(() => {
    if (status === 'idle') {
      dispatch(fetchTodos());
    }
  }, [status, dispatch]);

  const handleSubmit = (e) => {
    e.preventDefault();
    if (inputText.trim()) {
      dispatch(todoAdded(inputText)); // So simple!
      setInputText('');
    }
  };

  if (status === 'loading') {
    return <div>Loading todos...</div>;
  }

  if (status === 'failed') {
    return <div>Error: {error}</div>;
  }

  return (
    <div className="todo-app">
      <h1>RTK Todo App</h1>
      
      {/* Add new todo */}
      <form onSubmit={handleSubmit}>
        <input
          type="text"
          value={inputText}
          onChange={(e) => setInputText(e.target.value)}
          placeholder="What needs to be done?"
        />
        <button type="submit">Add Todo</button>
      </form>

      {/* Filter buttons */}
      <div className="filters">
        {['all', 'active', 'completed'].map(filterType => (
          <button
            key={filterType}
            className={filter === filterType ? 'active' : ''}
            onClick={() => dispatch(filterChanged(filterType))}
          >
            {filterType.charAt(0).toUpperCase() + filterType.slice(1)}
          </button>
        ))}
      </div>

      {/* Todo list */}
      <ul className="todo-list">
        {todos.map(todo => (
          <li key={todo.id} className={todo.completed ? 'completed' : ''}>
            <span
              onClick={() => dispatch(todoToggled(todo.id))}
              style={{ cursor: 'pointer' }}
            >
              {todo.text}
            </span>
            <button
              onClick={() => dispatch(todoDeleted(todo.id))}
              className="delete-btn"
            >
              ×
            </button>
          </li>
        ))}
      </ul>

      <div className="stats">
        <span>{todos.length} item(s) shown</span>
      </div>
    </div>
  );
}

export default TodoApp;

RTK Best Practices & Tips

✅ Do This

  • Keep slices focused: One slice per feature or domain
  • Use createAsyncThunk: For all API calls and async operations
  • Write "mutative" logic: state.count++ is totally fine!
  • Export selectors: Keep data access logic in slices
  • Use RTK Query: For server state management

❌ Don't Do This

  • Don't return state: Immer handles immutability
  • Don't put everything in Redux: Keep local state local
  • Don't make slices too big: Split large slices into smaller ones
  • Don't mix sync/async: Use extraReducers for async actions
  • Don't ignore DevTools: They're incredibly powerful!

🎯 Pro Tips for RTK Success

Naming Conventions:
  • • Slice names: camelCase (userProfile, shoppingCart)
  • • Action names: verb + noun (userAdded, cartCleared)
  • • Async thunks: verb + Resource (fetchUser, updateProfile)
File Organization:
  • • One slice per file
  • • Group related slices in folders
  • • Export selectors from slice files
  • • Keep store configuration separate

Migration Tips: From Redux to RTK

Easy Migration Strategy

  • • Start with new features using RTK
  • • Convert one slice at a time
  • • RTK is 100% compatible with existing Redux
  • • No need to rewrite everything at once
  • • Keep existing reducers while adding RTK slices
  • • Team can learn RTK gradually

What to Convert First

  • • Simple slices (like counter, toggles)
  • • New features you're building
  • • Slices with lots of boilerplate
  • • API-heavy slices (use RTK Query)
  • • Complex async logic (use createAsyncThunk)
  • • Leave working code alone until you need to change it

🚀 Migration Checklist

Install @reduxjs/toolkit
Replace createStore with configureStore
Convert one reducer to a slice
Update imports in components
Test everything works the same

🎯 RTK Migration Golden Rule

"If it ain't broke, don't fix it immediately." RTK works alongside traditional Redux perfectly. Migrate gradually, starting with new features, then convert old code when you need to modify it anyway.

Creating an API Slice

Define your API endpoints and automatically generate hooks for React components.

api.js 
import { createApi, fetchBaseQuery } from '@reduxjs/toolkit/query/react';

export const apiSlice = createApi({
  reducerPath: 'api',
  baseQuery: fetchBaseQuery({
    baseUrl: '/api',
    prepareHeaders: (headers, { getState }) => {
      const token = getState().auth.token;
      if (token) {
        headers.set('authorization', `Bearer ${token}`);
      }
      return headers;
    }
  }),
  tagTypes: ['Post', 'User'],
  endpoints: (builder) => ({
    getPosts: builder.query({
      query: () => 'posts',
      providesTags: ['Post']
    }),
    
    getPost: builder.query({
      query: (id) => `posts/${id}`,
      providesTags: (result, error, id) => [{ type: 'Post', id }]
    }),
    
    createPost: builder.mutation({
      query: (newPost) => ({
        url: 'posts',
        method: 'POST',
        body: newPost
      }),
      invalidatesTags: ['Post']
    }),
    
    updatePost: builder.mutation({
      query: ({ id, ...patch }) => ({
        url: `posts/${id}`,
        method: 'PATCH',
        body: patch
      }),
      invalidatesTags: (result, error, { id }) => [{ type: 'Post', id }]
    }),
    
    deletePost: builder.mutation({
      query: (id) => ({
        url: `posts/${id}`,
        method: 'DELETE'
      }),
      invalidatesTags: ['Post']
    })
  })
});

// Export hooks for usage in components
export const {
  useGetPostsQuery,
  useGetPostQuery,
  useCreatePostMutation,
  useUpdatePostMutation,
  useDeletePostMutation
} = apiSlice;

Using RTK Query Hooks

Use the generated hooks in your React components for automatic data fetching and caching.

PostsList.jsx 
import React from 'react';
import {
  useGetPostsQuery,
  useCreatePostMutation,
  useDeletePostMutation
} from './api';

function PostsList() {
  const {
    data: posts,
    error,
    isLoading,
    isFetching,
    refetch
  } = useGetPostsQuery();

  const [createPost, { isLoading: isCreating }] = useCreatePostMutation();
  const [deletePost] = useDeletePostMutation();

  const handleCreatePost = async () => {
    try {
      await createPost({
        title: 'New Post',
        content: 'Post content...'
      }).unwrap();
    } catch (err) {
      console.error('Failed to create post:', err);
    }
  };

  const handleDeletePost = async (id) => {
    try {
      await deletePost(id).unwrap();
    } catch (err) {
      console.error('Failed to delete post:', err);
    }
  };

  if (isLoading) return <div>Loading...</div>;
  if (error) return <div>Error: {error.message}</div>;

  return (
    <div>
      <button onClick={handleCreatePost} disabled={isCreating}>
        {isCreating ? 'Creating...' : 'Create Post'}
      </button>
      
      <button onClick={refetch}>Refresh</button>
      
      {isFetching && <div>Updating...</div>}
      
      <ul>
        {posts?.map(post => (
          <li key={post.id}>
            <h3>{post.title}</h3>
            <p>{post.content}</p>
            <button onClick={() => handleDeletePost(post.id)}>
              Delete
            </button>
          </li>
        ))}
      </ul>
    </div>
  );
}
